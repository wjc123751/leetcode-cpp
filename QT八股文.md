# QT 八股文
## 一、QT基础
### Qt的跨平台特性是如何体现的
1. QT给所有平台提供完全一样的接口
打开文件:QFile
窗口按钮:QPushButton
网络请求:QNetworkRequest
多线程:QThread
2. 底层调用系统原生接口
windows->win32 API
linux->x11/wayland
macos->Cocoa
3. 跨平台编译工具qmake/cmake
windows->msvc/mingw
linux->gcc
mac->clang
4. 统一的底层模块封装
线程/进程
文件IO
网络(Tcp/Udp/Http)
串口/usb
数据库
定时器、事件循环
图片、视频、音频
5. 一次编写，到处编译
### 上位机与下位机的关系是什么，你做过哪些相关项目？
上位机:电脑端应用软件，负责人机界面、数据展示、参数配置、下发控制指令；
下位机是单片机/嵌入式硬件，负责传感器和外设数据采集、底层硬件控制。
两者通过串口或网络协议双向通信
### 如何使用样式表更改一个进度条的外观？
使用QProgressBar创建进度条，通过Qt样式表QSS，分别对QProgressBar控制外框背景、边框圆角，用QProgressBar::chunk控制进度填充快颜色、渐变、圆角，自定义进度条外观
### 如何提升组件
使用raise函数来提升组件，将组件置于父组件的顶层，并且在同级组件中处于最前面位置，避免被其他控件遮挡
### QApplication的功能是什么？
是QT应用程序的入口管理类
1. 初始化QT框架
初始化QT环境
2. 管理事件循环
通过exec启动并维护主事件循环
3. 统一资源和全局配置
管理整个程序的公共资源
4. 信号槽和事件分发
将信号、事件分发给对应控件
5. 控制程序生命周期
6. 命令行参数解析
