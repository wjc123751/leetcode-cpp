# 双指针算法
1. 一句话总结：用两个指针，一遍遍历解决问题，把 O(n²) 降成 O(n)。
2. 作用：减少一层循环，不用开额外数组。
3. 本质：利用数组已经有序的特点，让两个指针往中间 / 两边走，快速缩小范围。

---

## 什么条件下才能用双指针？
满足下面任意一条，就可以考虑双指针：

1. 数组已经排序（最常见！）
   - 典型题目：三数之和、两数之和 II、接雨水、盛最多水的容器
   - 核心逻辑：
     - 和偏小 → 左指针右移
     - 和偏大 → 右指针左移

2. 要从两端往中间夹，找一对值满足条件
   例如：
   - 找和为 target 的两个数
   - 找和最接近的两个数
   - 找能盛水的最大面积
   - 去重、反转、滑动窗口

3. 要在一维数组上做“对撞”“快慢追赶”
   - 快慢指针（判断环、找中点）
   - 左右对撞指针（有序数组）

---

## 最典型的两类双指针

### 1. 左右对撞指针
- 一个在左：`left = 0`
- 一个在右：`right = n - 1`
- 往中间移动

- 三数之和
- 两数之和
- 盛最多水的容器
- 反转数组
- 有序数组去重

### 2. 快慢指针（龟兔赛跑）
- 快指针走两步
- 慢指针走一步

- 判断链表是否有环
- 找链表中点
- 找重复数

# Kadane 算法（最大子数组和）
一句话核心：前面的和是负数，就扔掉不要，从当前数重新开始；
前面的和是正数，就带着一起走，因为能让我更大。
1. 只用两个变量
cur：以当前位置结尾的最大子数组和
maxSum：全局最大值
2. 核心公式（背下来）
cur = max(nums[i], cur + nums[i]);
maxSum = max(maxSum, cur);
什么意思？
如果 cur + nums[i] 还不如 nums[i] 本身大
→ 说明前面那段是负数累赘，直接丢掉
否则就带着前面的和一起走

# 单调栈算法
单调栈 = 专门寻找：每个元素的「左边/右边第一个比它大 / 比它小」的元素
1. 作用
每个元素右边第一个比它大的数
每个元素右边第一个比它小的数
每个元素左边第一个比它大的数
每个元素左边第一个比它小的数
所有这类题，暴力是 O(n²)，单调栈 O(n) 秒杀！
2. 应用
最大矩形面积
每日温度
接雨水
直方图最大矩形
下一个更大元素 I / II
股票价格跨度

# 二分查找
1. 为什么是 while (left < right) 而不是 left <= right？
一句话答案：因为这个算法用的是「不断收缩区间，最终只剩一个元素」的思路，不是「搜索元素」的思路。
区别在这里：
① while (left <= right)
用于：寻找某个目标值是否存在 循环结束时：left = right + 1，区间空了
② while (left < right)
用于：寻找极值（最大 / 最小）循环结束时：left == right区间里只剩下最后一个元素 → 就是答案！
2. 为什么是 mid = left + (right-left)/2 而不是 (left+right)/2？
一句话答案：防止整数溢出！
left + right 会变成 40 亿超过 int 最大值（约 21 亿）→ 溢出 → 变成负数 → 程序崩溃！

# 回溯法
1. 回溯法是什么？走不通就回头，换一条路再走本质就是：暴力枚举所有可能的路径，找到所有合法答案。
2. 回溯法解决什么问题？
全排列（给你几个数，列出所有顺序）
组合（从 n 个数里选 k 个）
子集（找出所有子集）
路径（迷宫路径、棋盘路径）
分割（字符串分割成合法串）
N 皇后、解数独
3. 核心逻辑
做出选择 选一个数 / 一个方向，加入当前路径标记 “已使用”
递归进入下一层 继续做下一个选择
撤销选择（回溯）退回一步把刚刚选的东西删掉取消标记
4. 对比dfs
对比维度	   普通 DFS	                              回溯法
目的	      遍历、找某一个结果、统计数量	         穷举出所有完整合法方案
状态处理    一般遍历完就完事，不恢复现场	         必须手动撤销选择、恢复状态
典型场景	   二叉树遍历、省份数量、网格搜连通块	   全排列、组合总和、子集、分割、N 皇后
剪枝	      少量边界剪枝	                        大量提前剪枝，砍掉无效分支
所有回溯算法，底层实现都是 DFS 但不是所有 DFS，都是回溯
回溯 = DFS + 尝试构造完整解 + 选完就撤回 + 试遍所有可能
5. 对比排列和组合
排列：讲究顺序，能回头选，必须标记用过    从头选 + 用 used
组合：不讲究顺序，不能回头选，不用标记    从 start 往后选 + 不用 used
6. 组合 vs 组合总和
组合：每个数字只能用 1 次 → 下一层递归 i+1
组合总和：每个数字可以无限用 → 下一层递归 i
7. 组合 vs 电话号码组合
组合：从一堆数里不断往后选，选够 k 个结束
电话号码：按顺序一层一层处理数字，每层选一个字母，处理完所有数字结束

# LRU
1. LRU = Least Recently Used：最近最少使用
2. 核心思想：淘汰掉 最久没被用过 的那个数据优先保留 最近刚用过 的数据
# LRUCache 
1. 一个固定容量的缓存容器：
有最大容量，满了还要再加新数据
自动把最久没访问过的删掉
最近查询、修改、新增过的，都放到最前面保护起来
2. 两个核心操作
get(key)
有这个 key：返回值，并且标记为最近使用
没有：返回 -1
put(key, value)
key 已存在：更新 value，标记为最近使用
key 不存在：
缓存没满：直接加进去
缓存已满：删掉最久没使用的，再加新的

# Trie (前缀树)
1. 字典树又称前缀树或Trie树，是一种哈希树的变种树形数据结构
Trie 是一种用于高效存储和检索字符串的多叉树结构，核心特点是共享公共前缀，大幅节省空间并提升查询效率。
2. 核心操作
insert(word) 插入单词
从根节点出发，遍历单词的每个字符
若当前字符对应的子节点不存在，则创建新节点
移动到该子节点，继续处理下一个字符
单词遍历完成后，将当前节点的 isEnd 标记为 true（表示这是一个完整单词）
search(word) 查找完整单词
从根节点出发，遍历单词的每个字符
若任意字符对应的子节点不存在，直接返回 false
遍历完成后，检查当前节点的 isEnd 是否为 true
为 true：说明存在该完整单词，返回 true
为 false：说明该路径只是其他单词的前缀，返回 false
startsWith(prefix) 查找前缀
逻辑和 search 几乎一致，区别在于：遍历完所有字符后，不需要检查 isEnd，只要路径存在，就说明有单词以该前缀开头，直接返回 true。
