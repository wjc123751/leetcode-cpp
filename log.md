# 双指针算法
1. 一句话总结：用两个指针，一遍遍历解决问题，把 O(n²) 降成 O(n)。
2. 作用：减少一层循环，不用开额外数组。
3. 本质：利用数组已经有序的特点，让两个指针往中间 / 两边走，快速缩小范围。

---

## 什么条件下才能用双指针？
满足下面任意一条，就可以考虑双指针：

1. 数组已经排序（最常见！）
   - 典型题目：三数之和、两数之和 II、接雨水、盛最多水的容器
   - 核心逻辑：
     - 和偏小 → 左指针右移
     - 和偏大 → 右指针左移

2. 要从两端往中间夹，找一对值满足条件
   例如：
   - 找和为 target 的两个数
   - 找和最接近的两个数
   - 找能盛水的最大面积
   - 去重、反转、滑动窗口

3. 要在一维数组上做“对撞”“快慢追赶”
   - 快慢指针（判断环、找中点）
   - 左右对撞指针（有序数组）

---

## 最典型的两类双指针

### 1. 左右对撞指针
- 一个在左：`left = 0`
- 一个在右：`right = n - 1`
- 往中间移动

- 三数之和
- 两数之和
- 盛最多水的容器
- 反转数组
- 有序数组去重

### 2. 快慢指针（龟兔赛跑）
- 快指针走两步
- 慢指针走一步

- 判断链表是否有环
- 找链表中点
- 找重复数

# Kadane 算法（最大子数组和）
一句话核心：前面的和是负数，就扔掉不要，从当前数重新开始；
前面的和是正数，就带着一起走，因为能让我更大。
1. 只用两个变量
cur：以当前位置结尾的最大子数组和
maxSum：全局最大值
2. 核心公式（背下来）
cur = max(nums[i], cur + nums[i]);
maxSum = max(maxSum, cur);
什么意思？
如果 cur + nums[i] 还不如 nums[i] 本身大
→ 说明前面那段是负数累赘，直接丢掉
否则就带着前面的和一起走

# 单调栈算法
单调栈 = 专门寻找：每个元素的「左边/右边第一个比它大 / 比它小」的元素
1. 作用
每个元素右边第一个比它大的数
每个元素右边第一个比它小的数
每个元素左边第一个比它大的数
每个元素左边第一个比它小的数
所有这类题，暴力是 O(n²)，单调栈 O(n) 秒杀！
2. 应用
最大矩形面积
每日温度
接雨水
直方图最大矩形
下一个更大元素 I / II
股票价格跨度

# 二叉树 先 / 中 / 后序 栈 + 迭代
外层循环永远一样：cur != nullptr || !st.empty() 左遍历永远一样 区别只有：访问时机 + 出栈时机
遍历	访问时机	出栈时机	      代码位置
先序	入栈前	左走完就出栈	内层 while 里
中序	出栈时	左走完就出栈	左走完后
后序	右走完后	访问后才出栈	需要 pre 判断
