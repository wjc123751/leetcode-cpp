# C++ 八股文
## 一、C++基础
### 讲一讲封装、继承、多态是什么
封装:将数据和操作数据的方法放在一起，对外留出接口
继承:子类复用父类的代码，扩展新功能
多态:一个接口，不同子类有不同实现
### 多态的实现原理 以及 多态的优点
动态多态(运行时多态):通过虚函数实现，一个类只有一张虚函数表，每个对象有一个虚函数指针。父类指针或引用指向子类对象时，运行时通过对象的虚函数指针找到虚函数表，调用子类重写的函数，实现运行时多态。
静态多态(编译时多态):通过函数重载或模板实现，函数重载是函数名相同，参数个数、类型。顺序不同；模板一套代码适配任意类型。
优点:用父类的统一接口，子类进行功能扩展
### final 标识符的作用是什么？
修饰类不能继承；修饰虚函数不能重写；修饰变量不能修改。
### 虚函数是怎么实现的？它存放在哪里在内存的哪个区？什么时候生成的
有虚函数的类，编译器自动生成一张虚函数表，对象有虚函数指针指向虚函数表，父类指针或引用指向子类对象时，通过子类对象的虚函数指针找到虚函数表，调用子类重写的函数。
虚函数和普通函数一样在代码区
编译阶段生成
### 智能指针的本质是什么，它们的实现原理是什么？
本质是封装原始C++指针的类
原理是通过对象生命周期和析构函数自动释放堆内存，通过引用计数实现共享
### 匿名函数的本质是什么？优点是什么？
本质是编译器自动生成的匿名仿函数类，实例化为临时对象，重载()运算符，可以进行函数调用
优点是不要函数声明和定义，用完自动销毁，节省空间
### 右值引用是什么，为什么要引入右值引用？
是给临时变量取别名
为了支持移动语义，避免临时变量频繁拷贝，还能实现完美转发
### 左值引用和指针的区别？
初始化:指针可以不初始化，引用必须初始化
性质不同:指针是变量，引用是给对象取别名
占用内存空间不同:指针有自己的内存空间，引用和被引用对象占同一空间
### 指针是什么？
指针是用来存放地址的变量
### weak_ptr真的不计数？是否有计数方式，在哪分配的空间
计数，但是不影响强引用计数use_count(不影响对象生命周期)，会操作弱引用计数weak_count(管理控制块生命周期);
计数存在堆上共享控制块，weak_ptr只在栈上存一个控制块指针;
shared_ptr 如果是使用make_shared初始化，控制块和对象在同一块堆空间
shared_ptr 如果是用new构造shared_ptr，控制块和对象在两块堆空间
### malloc的内存分配的方式有几种，有什么缺点？
有两种
当用户申请内存小于128KB通过brk()系统调用从堆分配内存
当用户申请内存大于128KB通过mmap()系统调用从文件映射区分配内存
缺点:容易造成内存泄漏和内存碎片，要判断内存是否分配成功，内存释放后还要将指针赋值为空避免野指针
### 为什么不全部使用mmap来分配内存？
mmap是系统调用，执行系统调用要从用户态进入内核态，执行完后返回用户态，状态的切换要耗费很多时间。
mmap申请的内存每次释放时，都要归还操作系统，再次申请的虚拟地址处于缺页状态，第一次访问该虚拟地址会触发缺页中断。
虽然brk也是系统调用，但是brk一次性扩展一大片堆内存，后续小内存分配、释放都在用户态内部空闲链表完成，不用频繁执行系统调用
### 为什么不全部都用brk
brk管理的堆，是一块连续的虚拟内存
全部用brk申请内存，随着程序频繁malloc和free，堆内将产生越来越多的不可用的外部内存碎片
### free传入一个指针，它如何确定具体要清理多少空间呢
申请内存时，会多分配16字节内存，里面保存了内存块的详细信息(内存块的大小、状态、链表信息)
free会对传入的内存地址向左偏移16字节，获取内存块详细信息
### define和const的区别是什么？
阶段:define在预处理阶段进行简单文本替换;const在编译阶段确定其值
安全性:define没有数据类型，不会进行类型安全性检查;const有类型，会进行类型判断
内存占用:define纯文本直接替换，没有独立内存空间;全局、静态const在数据区，局部const在栈区
调试:define不能调试;const可以调试
### 程序运行的步骤是什么
预处理:处理头文件，进行宏替换，生成.i文件
编译:做词法分析、语义分析、错误检查，将源码翻译成汇编语言，生成.s文件
汇编:汇编器将汇编语言翻译成机器语言，生成.o文件
链接:将目标文件和库文件链接到一起，生成可执行文件.exe
运行:可执行文件加载到内存，创建进程，CPU执行指令(断点调试 gdb调试)
### 程序生命周期
编译阶段 链接阶段 加载阶段 运行阶段
### 静态编译和动态编译分别是什么
静态编译:链接时把库代码打包进程序，独立运行，exe体积大，库不共享
动态编译:链接时只做标记，运行时加载外部库，exe体积小，库可共享，依赖环境
### 什么是原子操作
指不会被线程调度机制打断的操作，这种操作一旦开始，就一直运行到结束，中间不会切换到另一线程
### 锁的底层原理是什么？
利用CPU硬件提供的原子指令，保证同一时刻只有一个线程进入临界区，实现并发互斥
CAS机制:三个参数(内存地址V, 旧预期值A, 新值B) 读取内存地址V当前值，比较当前值和旧预期值，相等则更新成新值B，返回成功；不相等则不做修改，返回失败
CAS机制就是自旋锁和互斥锁的底层实现方式
自旋锁 = 死循环 + CAS
while(CAS(lock, 0, 1)){}
抢不到就原地自旋轮询，不放弃CPU
互斥锁 = 自旋锁 + 休眠/唤醒线程
先CAS自旋抢一会，抢不到，线程阻塞休眠，让出CPU交给内核调度；锁释放后，唤醒线程，再用CAS抢锁
### class与struct的区别
默认继承权限不同:class默认继承private权限;struct默认继承public权限
### 内存对齐是什么？为什么要进行内存对齐？内存对齐有什么好处？
内存对齐:把变量的内存地址按照特定整数倍数存放
CPU读取内存是按照块批量读取；兼容不同指令集架构的CPU
减少内存访问次数
### 进程之间的通信方式有哪些？
管道:半双工通信，一个进程读，一个进程写 不适合进程间频繁交换数据
消息队列:可以边发边收，支持按消息类型收发 存在用户态和内核态数据拷贝问题
共享内存:速度最快 解决消息队列用户态和内核态数据拷贝问题
信号量:用于多进程间的同步、互斥，保护临界区资源
信号:异步软中断，用于进程通知、终止、异常处理
套接字:支持本机和跨网络通信，TCP和UDP都是基于套接字实现
### 线程之间的通信方式有哪些？
信号量:用计数器管理资源数量
条件变量:搭配互斥量使用，让线程等待/唤醒
互斥量:保证同一时刻只有一个线程访问临界区资源
### 介绍一下socket中的多路复用，及其他们的优缺点，epoll的水平和边缘触发模式
单线程通过一个系统调用，同时监听多个socket文件描述符，只要有任意一个文件描述符异常，就返回并处理
三种实现:select poll epoll
1. selete
把要监听的fd放入fd_set位图集合，拷贝到内核，内核遍历所有fd检测就绪状态，返回就绪的fd集合，用户态再遍历找就绪fd集合处理
优点:windows/linux都支持
缺点:最大文件描述符有上限1024，高并发不够用；每次调用都要拷贝fd集合到内核，开销大；用户态内核态都要遍历所有fd
2. poll
用struct pollfd数组代替位图，监听fd、事件类型，内核遍历检测就绪
优点:突破selete 1024 fd上限
缺点:每次调用都要拷贝fd集合到内核，开销大；用户态内核态都要遍历所有fd
3. epoll
在内核维护红黑树+就绪链表
优点:无fd数量上限；找就绪fd时间复杂度O(1)，只遍历就绪fd；只需要一次注册，不用每次拷贝所有fd
缺点:仅linux支持
epoll水平触发(Level Trigger)模式:有数据就一直通知
epoll边缘触发(Edge Trigger)模式:状态变化时会通知一次，需要非阻塞IO循环读取数据
### 类的生命周期
从加载到内存开始，到卸载出内存为止
包括加载、验证、准备(静态变量分配内存、赋默认值)、解析(符号引用->直接内存地址引用)、初始化(给静态代码块、静态变量赋值)、使用、卸载
类的初始化只会执行一次
静态变量:类加载和初始化阶段赋值
实例化成员变量:new对象的时候赋值
### 父类的构造函数和析构函数是否能为虚函数？这样操作导致的结果？
父类的构造函数不能为虚函数:调用虚构造函数需要虚指针，但是虚指针要等构造函数执行完才能初始化
父类的析构函数可以且经常为虚函数:继承情况下，父类析构函数必须设为虚函数，不然当父类指针指向子类对象时，只会调用父类析构函数，子类内存泄漏
### 多线程为什么会发生死锁，死锁是什么？死锁产生的条件，如何解决死锁？
多进程对资源进行竞争，如果一个进程集合的每个进程都在等待这个集合中的另一个进程才能继续往下执行，没有外力无法继续执行，这就是死锁
条件:互斥 请求和保持 不可剥夺 环路等待
破坏其中一个条件
### 描述一下面向过程和面向对象
面向过程:将问题分析出解决问题的步骤，将步骤一个一个实现，逐个调用
面向对象:将问题分解为各个对象，建立对象是为了描述某个事物在整个解决问题的步骤中的行为
### C++中左值和右值是什么？++i是左值还是右值，++i和i++哪个效率更高？
左值:有内存地址，能取地址
右值:临时值，无固定内存地址，不能取地址
++i是左值
++i返回的是左值，没有发生拷贝，效率高
### 介绍一下vector、list的底层实现原理和优缺点
vector 底层由一块连续的内存空间组成，由三个指针实现的分别是头指针start、尾指针finish、可用空间尾指针end_of_storage
内存增长规则:当finish == end_of_storage时，申请2倍大小新连续内存，拷贝数据，释放旧内存，更新三个指针
优点:可以下标随机访问，尾插尾删效率高
缺点:前面部分插入删除效率低；扩容效率低
list 底层双向链表实现
优点:按需申请内存，不要扩容，任意位置插入删除效率高
缺点:不支持下标访问
### 静态变量在哪里初始化？在哪一个阶段初始化？（都存放在全局区域）
静态变量、全局变量、常量都是在编译阶段完成初始化和内存分配；其他变量都是编译阶段完成初始化，运行阶段完成内存分配
### 如何实现多进程？
linux中C++使用fork函数创建进程
windows中C++使用createprocess来创建进程
### 空对象指针为什么能调用函数？
类的普通成员函数和静态成员函数，编译后都放在类的外部，所有对象共享一份代码，节省内存；
普通成员函数自带隐式this指针，调用时把对象指针传给this；空对象指针调用时this为空，只要函数内部不访问成员变量，就不会解引用空指针，可以正常执行；
静态成员函数没有this指针，与对象无关，也可以空指针调用；
### shared_ptr线程安全吗？
shared_ptr的引用计数是线程安全的，但是指向的对象不是线程安全的，指针自身赋值、修改指向不是线程安全的
### push_back()左值和右值的区别是什么？
如果push_back()传左值，则使用它拷贝构造新对象
如果push_back()传右值，则使用它移动构造新对象，不拷贝效率高

### 智能指针有哪些？怎么使用？
三种 unique_ptr shared_ptr weak_ptr
1. unique_ptr 独占所有权，不能拷贝，只能移动
2. shared_ptr 引用计数机制，每多一个指针引用，计数+1，指针销毁，计数-1，计数为0自动释放内存
3. weak_ptr 配合shared_ptr使用，解决循环引用造成内存泄漏
unique_ptr<int> up = make_unique<int>(100);
unique_ptr<int> up2 = move(up);
shared_ptr<int> sp1 = make_shared<int>(200);
cout << sp1.use_count();
shared_ptr<int> sp2 = sp1;
weak_ptr wp = sp1;
shared_ptr tmp = wp.lock();
if(tmp){
    cout << *tmp;
}
4. 循环引用问题
两个shared_ptr管理的类对象，互相持有对方的shared_ptr，你指着我，我指着你，形成环形依赖，引用计数永远减不到0，造成内存泄漏。
解决办法:用weak_ptr代替其中一个shared_ptr
### 匿名仿函数类是什么？
编译器给lambda匿名函数自动生成的、没有类名、重载了()运算符的临时类。
auto f = [](int a){ cout << a; };
等价于
class 无名类名{
public:
    void operator()(int a) const{
        cout << a;
    }
};
无名类名 f;
### 右值引用&&怎么用在移动语义和完美转发的？
1. 移动语义:偷资源，不拷贝
// 移动构造
String(String&& other){
    data = other.data;
    other.data = nullptr;
    cout << 移动构造;
}
// 传临时右值触发移动构造
String s2(String("world"));
// 左值强转右值，触发移动构造
String s3(move(s1));
2. 完美转发:保留左值右值属性，原样转发
左值:有名字，能取地址
右值:没名字，临时值
不要完美转发，不管传左值还是右值，到了内层全变成左值，没法匹配重载的右值版本函数
void func(int& x){cout << "左值:" << x;}
void func(int&& x){cout << "右值:" << x;}
template<typename T>
void wrapper(T&& t){
    func(forward<T> t);
}
int a = 10;
wrapper(a);
wrapper(20);
### 阻塞IO和非阻塞IO
阻塞IO:没有数据就阻塞等待，直到有数据才返回
非阻塞IO:有没有数据都返回，需要不断轮询有没有数据，但是一个线程可以轮询多个连接
### this指针
常量指针 指针指向不能改变，指针指向的值可以改变
this指针是类的非静态成员函数隐含的第一个形参，指向当前调用这个函数的对象本身
三大作用
1. 区分成员变量和局部重名变量
2. 返回当前对象本身，支持链式调用
3. 在成员函数中访问当前对象成员
this就是当前对象的地址，由编译器隐式传入非静态成员函数
### 线程安全
多线程同时访问、操作同一份代码或数据，不会出现数据竞争、逻辑错乱就是线程安全
